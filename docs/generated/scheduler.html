scheduler¶

scheduler: Use a nearest-neighbor variant to obtain an observation schedule

Function scheduler is the main tool in this software. The aim is to obtain an observation schedule for a high-cadence telescope, where the objective is to maximize the number of positions in the sky, from a given list, which are observed at least three times.

See simulator.py or main part below for examples of usage.

Obtain a schedule based on a variant of the nearest-neighbor heuristic

Repeatedly construct observation schedules, until time_lim is reached. Each construction is based on a simulation, where at each point in time an observation is chosen among obs_data.sky positions available which are closest to the current position of the telescope.

Return type

object of class Solution, specifying observation sequence and its evaluation

Parameters

  • obs_data – data specifying the instance to be solved: sky positions and start/end times

  • time_now – current time

  • last_obs – sequence of observations made before

  • earliest – dictionary, position k cannot be observed before earliest[k]

  • current_best – previously best known solution (None if not available)

  • hidden_list – indices in obs_data.sky that cannot be observed now (see hidden.py)

  • time_lim – clock time allowed for obtaining a solution (in seconds)

Returns

best sequence of observations found
